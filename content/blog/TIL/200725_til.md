### Week 12 - AI모델에게도 예비 고사와 수능이 있다고요?

#### 교차 검증 (Cross-Validation)

모델을 학습시킬 때는 학습 데이터(Training Data)뿐만 아니라 평가 데이터(Evaluation Data)도 필요함.

주로 전체 데이터 중 10% ~ 30%를 평가 데이터로 할당을 하는데 이렇게 전체 데이터 중에서 일부분을 떼어 평가 데이터로 사용하는 방식을 Holdout Method라 함.

평가 데이터를 Validation과 Test 두 가지로 세트로 나눌 수 있음. Validation Set는 모의 고사, Test Set은 수능으로 비유를 들 수 있다.

Validation set을 사용하는 이유는 아래와 같다.

- 어떤 모델 구조, 크기를 사용할지, 모델 변수들은 무엇으로 할지 결정 (Model Hyperparameter)

- 얼마나 오래, 빠르게 학습을 시킬 것인지 결정 (Learning Hyperparameter)

### 모델을 잘 학습시키는 팁

1. 학습과 평가 데이터를 무작위(Random Shuffle)로 잘 나누고 있는지 확인
2. 평가 데이터와 학습 데이터가 중복이 되지 않는지 확인
3. 세 부분의 분포가 최대한 비슷하게 하기
4. 데이터 수가 적으면 K-fold Cross Validation 이용
5. I would like to emphasize is the importance of not looking at a held-out test set - 조경현 교수님
